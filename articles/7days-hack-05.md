## Gobuster

Go-lang製のブルートフォース攻撃ツール。高速に動作するのが売り。

以下のようなコマンドで攻撃を仕掛けることが出来る。
使い方は `dirb` コマンドとほぼ同じ。

```sh
$ gobuster dir -u http://{IP}:{Port} -w {PathToWordList}
```

## ディレクトリ リスティング 機能

`Directory Listing`

Webサーバが、特定のディレクトリにインデックスファイル (`index.html`など) が存在しない場合、
そのディレクトリ内のファイルやサブディレクトリのリストを自動的にブラウザに表示する機能。

意図せずファイルやディレクトリ構造が外部に公開されるため、セキュリティ上あまり宜しくない。

## リバースシェル

`Reverse Shell`

`ターゲットマシン→ローカルマシン`へ向けた接続を行う仕組み。
通常なら、`ローカルマシン→ターゲットマシン`へ接続を行ってターゲットマシンを操作するが、その逆。

ローカル→ターゲット(外→内)の経路は通常ファイアウォールで塞がれていることが多いが、
ターゲット→ローカル(内→外)の経路は出入り自由なことが多いため、悪用するにはもってこい。

リバースシェル自体は悪意があるものではないが、悪用されてしまうことがある。

## SUID (Set User ID)

誰が実行しても所有者の権限で実行できるような、特殊な権限のこと。

例えば、`passwd` コマンドの実行の権限を見てみると、`rws`とある。

```sh
$ ls -l `which passwd`
-rwsr-xr-x 1 root root 68208 Feb  6  2024 /usr/bin/passwd
```

この `s` がSUID権限で、`passwd`コマンドは必ず所有者`root`の権限で実行される。

以下のコマンドで、Rootフォルダから、SUID権限がついているバイナリを探索することが出来る。

```sh
find / -perm -u=s -type f 2>/dev/null
```
